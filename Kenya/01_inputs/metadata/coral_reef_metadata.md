## SEEA EA Relevance

- **Condition accounts (coral reef M1.3):** Fish UVC (Files 1, 6, 7) and coral health condition sheets (Files 3, 8) are primary inputs for biotic condition indicators; coral size structure supports structural condition indices
- **Extent accounts:** Benthic cover sheets (File 8 Extent; Files 4, 5) provide ground-truth data for satellite-derived reef extent mapping
- **Recruitment / recovery:** Recruit sheets (Files 3, 8) support coral recovery rate indicators
- **Multi-year potential:** File 5 spans 2024-2025; File 7 is structured for annual updates — enables change detection between accounting periods
- **Data gaps:**
  - No invertebrate data from COMRED July 2025 campaign (sheet present but empty in File 6)
  - Fish size classes recorded as range strings (e.g., "5--10") in UVC files; conversion to midpoint values needed for biomass calculations
  - No a/b allometric parameters included; length-weight parameters must be sourced from FishBase
  - Transect area assumptions need verification (File 1: 50 m transects, width not specified; File 2: 25 m)
  - No trophic/functional group assignments in fish data (unlike Madagascar dataset)

## Data Quality Flags

- **Location name inconsistencies across files:** "Sultance palace" (Files 1, 2) vs "Sultan palace" (other references); "Kilifi Wesa" (File 3) vs "Wesa" (other files); standardise before merging
- **Duplicate longitude column:** File 1 has two longitude columns (longtitude + Longtitude); one appears to be a copy — verify before use
- **Latitude sign errors:** File 2, row 2 has positive latitude (3.94326) vs negative in other rows for same site; verify sign convention
- **Template rows in KLF consolidated file:** File 7 contains pre-populated template rows for years 2025-2092 with no observation data; filter to actual 2024 data only before analysis
- **File 4 vs File 5 overlap:** CORAL COVER (File 4) appears to be a subset of CORAL-DATA (File 5); confirm which is the authoritative version before use
- **Species spelling variation in fish files:** Minor inconsistencies in species names (e.g., "parupeneus macronemus" vs "Parupeneus macronemus"); case standardisation needed
